# 1. Краткий   

```
Общее время: {N} мин
Этап работ: {Диагностика / Диагностика} / {Настройка / Консультация} / {...}
Конкретная задача: {суть проблемы из заявки}
Выполненные действия: {что сделали, по шагам}
Фиксация изменений: {что меняли; если параметры — до -> после}
Проверка результата: {как проверили; сколько тестов; какие метрики}
Результат: {измеримый или проверяемый результат}
Причина: {найденная первопричина}
```

**Пример (камера):**
```
Общее время: 60 мин 
Этап работ: Диагностика / Настройка
Конкретная задача: Камера не распознаёт товар на линии
Выполненные действия: проверка освещения и положения камеры; настройка экспозиции / порога распознавания
Фиксация изменений: экспозиция 40 -> 65; threshold 120 -> 95
Проверка результата: 20 тестовых сканов; сравнение процента пропусков до и после
Результат: количество пропусков снизилось с 14% до 2%
Причина: недостаточная экспозиция камеры при изменении освещения
```

# 2. Развёрнутый

```
Проблема: {одной фразой}
Симптомы:
— {симптом 1}
— {...}
Общее время: {N} мин
Этап работ: {Диагностика / Диагностика} / {Настройка / Консультация} / {...}
Результат: {итог с цифрами или фактом}
Причина: {найденная первопричина}

Что проверяли:
— {пункт 1}
— {...}
Что изменили:
— {параметр: значение до → после}
— {...}
Как проверяли:
— {метод проверки}
— {...}

Время этапа работы: 
— {Этап} {A} мин
— {...} {...} мин

Статус: {решено / в работе / эскалация}
Теги: {#тема1 #тема2 #тема3}
```

**Пример (камера):**
```
Проблема: Камера не распознаёт товар на линии
Симптомы:
— пропуски объектов
— нестабильное распознавание
— рост брака
Общее время: 60 мин
Этап работ: Диагностика / Настройка
Результат: количество пропусков снизилось с 14% до 2%
Причина: недостаточная экспозиция камеры при изменении освещения

Что проверяли:
— освещение линии
— положение камеры
— параметры threshold
— качество кадров
Что изменили:
— экспозиция: 40 -> 65
— threshold: 120 -> 95
Как проверяли:
— 20 тестовых сканов
— сравнение до/после
— анализ процента брака

Время этапа работы: 
— Диагностика 40 мин
— Настройка 20 мин

Статус: Решено 
Теги: #камера #распознавание #экспозиция #vision #брак 
```

# Разница в формах отчёта
### Добавлены поля:
1. **Конкретная задача** разделено на **проблема** + **симптомы**.
2.  **Время этапа работы** - разделённое общее время.
3. **Статус** - статус в котором находиться заявка - *авто*.
4. **Теги** - теги для быстрого нахождения схожих заявок (под поиск) - *авто*.

### Поля переименованы для простоты:
1. **Выполненные действия** -> **Что проверяли**
2. **Фиксация изменений** - > **Что изменили**
3. **Проверка результата** -> **Как проверяли**

### Оформление
1. В кратком шаблоне первым идёт время + тип работы, в развернутом шаблоне идут основные характеристики заявки (проблема -> симптом -> время -> этап -> причина) - сразу видна проблема - решение.
2. Поменян общий вид оформления, где каждый подпункт идёт с новой строки, для простоты чтения.

# Что нужно
Фиксация в отчёт симптомов - как клиент человеческим языком озвучивает проблему, то есть разделить **Конкретную задачу** на: **проблема** + **симптом**.

# Подсказки по заполнению (типовые темы DMC)

| Тема заявки | Тип работы | Что указать в «изменениях» | Что указать в «проверке» |
|-------------|------------|----------------------------|--------------------------|
| Задание / ТСД | Диагностика / Изменение в DMC | № задания, статус до -> после | задание снова на ТСД, клиент запустил |
| Коды / агрегация | Работа с кодами / БД | какие коды, какой статус изменён | сверка в DMC и ЧЗ |
| Принтер / печать | Настройка принтера | шаблон, прошивка, датчики | тестовая этикетка, читаемость кода |
| ЧЗ / оборот | Работа с ЧЗ | документ, отчёт, статус кодов | документ принят, коды в нужном статусе |
| Консультация | Консультация | что объяснили клиенту | клиент подтвердил понимание / выполнил шаг |
